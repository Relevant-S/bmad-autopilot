## Concurrency-overlap witness

`max_parallel_stories=2` with exactly two `ready-for-dev` stories means both are admitted in a single wave. Each injected runner body records its entry (a thread-safe peak-in-flight counter), then rendezvous at a `threading.Barrier(2)`. The barrier RELEASING is positive proof both runner bodies executed simultaneously — a sequential dispatcher would block forever at the barrier. The rendezvous is **loud-fail bounded**: `barrier.wait(timeout=5.0)` raises `BrokenBarrierError` (and breaks the barrier) on timeout, so a regression to sequential dispatch surfaces as a fast, hard test FAILURE — never a CI hang (never an unbounded `Barrier.wait()`). The fixture asserts the peak observed-concurrency counter reached 2 (an explicit assertion, not merely a non-hang). Completion order is non-deterministic, so `dispatched_story_ids` is asserted set-wise; the cost-partition sums and the terminal state ARE deterministic (the folds are commutative + single-writer on the main thread).

## Substrate defect + fix (surfaced by this integration witness)

This run is the FIRST to drive the real `run_epic_loop(parallel_stories=True)` against a real git repo with the real `ParallelEnvClaimProvider` (every Story 18.1–18.3 unit test stubbed `create_worktree` / `worktree_run_state_path`, so none exercised the genuine provider↔worktree ordering). It surfaced a latent seam defect: `ParallelEnvClaimProvider.__call__` eagerly called `pre_seed_parallel_env` on the main thread during admission, writing `run-state.yaml` INTO the per-worktree directory **before** the worker thread's `create_worktree` ran — so `git worktree add` failed against the now-non-empty target (`fatal: '<path>' already exists`). Per the loud-fail / surface-don't-paper-over doctrine, the defect was **fixed** (not worked around) as a contract-pair within Story 18.4: a new `ClaimCarrierSeed` seam (`parallel_dispatch`) defers the carrier write to `ParallelEnvClaimProvider.seed_carrier`, which the dispatcher invokes AFTER `create_worktree` (worktree exists) and before the runner (so the seeded disjoint port is on disk when env-provisioning reads it). `run_epic_loop`'s parallel branch now passes `claim_seed=claim_provider.seed_carrier`. The eager in-`__call__` pre-seed was removed. This is precisely the value of an end-to-end reference fixture: it caught a composition defect the unit tests structurally could not.
